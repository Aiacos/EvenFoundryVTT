<objective>
Create the standalone `HudDeltaDriver` class (new file `engine/hud-delta-driver.ts`) that
replaces the Phase 20 naive recomposite trigger with an event-driven, debounced, per-tile
xxhash delta loop. The driver owns: a one-time `await xxhash()` WASM init, a 4-slot
`prevHashes` table, a `setTimeout`-based debounce (configurable, default 100ms), multi-channel
WS subscriptions (`character.delta` + `combat.turn` + `combat.state`), a `runFirstFrame()` that
pushes all 4 tiles and seeds baseline hashes, and a `_runCycle()` that pushes only the tiles
whose h32 hash changed.

This plan is TDD: the test file `hud-delta-driver.test.ts` (Wave 0 scaffold) is written first
with the criterion-3 spy assertions, then the class is implemented to make them pass. The driver
is NOT yet wired into `LayerManager` here — that is Plan 24-02 (Wave 2).

Purpose: Eliminate all BLE pushes on an idle HUD (RPROMO-01) and collapse burst delta events
into one render cycle, while keeping per-tile re-encode selective.
Output: `engine/hud-delta-driver.ts` (HudDeltaDriver + DEFAULT_MIN_REDRAW_INTERVAL_MS) and its
unit test covering DL-01..DL-06 + first-frame.
</objective>
