<objective>
Lay the two foundations the canvas Status HUD depends on: (1) the VT323 font
loader with a tested monospace fallback chain (RFONT-01), and (2) the
`CanvasLayer.attachCanvas()` async-signature change (resolved Open Question Q1)
so a layer can finish loading VT323 + pre-baking chrome before its first paint.

Purpose: Without an awaitable `attachCanvas`, a layer cannot guarantee the font
and chrome bitmap exist before the first composite — the locked decision and SC1
require resolution "before the first frame". The font loader is the unit that
provides that resolution.
Output: `@fontsource/vt323` installed in g2-app; `ensureVt323Loaded()` +
fallback test; `CanvasLayer.attachCanvas` returns `Promise<void>`; CanvasCompositor
+ LayerManager call sites await it; zero test regressions.
</objective>
