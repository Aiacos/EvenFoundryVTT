<objective>
Wire the Deepgram Nova-3 Multilingual session to receive the keyterm-prompting vocabulary computed by plan 15-01's `buildKeytermList()`. The Deepgram adapter is extended with a `keytermProvider` callback that is invoked once per `connect()` call — the callback returns the current merged keyterm list, and the adapter appends one `keyterm=<URL-encoded>` query param per entry to the WS URL.

A callback (rather than a static array) is required so that plan 15-03's hot-update path can serve fresh keyterms on the next session-create without re-instantiating the adapter. For session-mid refresh, plan 15-03 also adds a `refreshKeyterm()` API; that is out of scope for this plan.

VOICE-06 is the headline REQ — the Deepgram URL must contain `keyterm=` query params. VOICE-08 is covered automatically because the merger output already interleaves IT + EN. CONTEXT D-02 (single Nova-3 Multilingual session fed both locales) is satisfied because the existing `language=multi` URL param remains unchanged.

Phase 12's existing test suite (DG-01..DG-13) must continue to pass — the keyterm wiring is purely additive (zero behaviour change when `keytermProvider` returns []).

Purpose: VOICE-06 keyterm parameter integration in `deepgram-stt.ts`. Output: extended `CreateDeepgramSttOpts`, updated URL builder, server.ts wiring at step 10, and 6 new test cases DGKT-01..06.

</objective>

Deepgram keyterm wire format (RESEARCH.md §2 Option C — verified 2026-05-17):
- One occurrence per term: `keyterm=fireball&keyterm=palla%20di%20fuoco`
- URL-encoding via `encodeURIComponent()` (Node native, RFC 3986 compliant). Multi-word IT spell names contain spaces — must be %20-encoded, NOT '+'.
- Empty keyterm list → omit the `keyterm` param entirely (preserves Phase 12 baseline transcription behaviour for users without entity-pack push).
