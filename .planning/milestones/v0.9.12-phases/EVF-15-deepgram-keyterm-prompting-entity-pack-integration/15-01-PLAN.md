<objective>
Build the pure vocabulary-merger module that produces the Deepgram `keyterm` list. This plan has zero SDK dependencies and zero side effects — it is a deterministic function over (static spell vocab, dynamic entity-pack snapshot) that returns a deduplicated, capped, locale-merged `string[]` ready to be appended as `keyterm=…` URL query parameters by plan 15-02.

Static SRD spell vocabulary is extracted into `@evf/shared-protocol` (data, not logic — belongs in the shared package) so the bridge can import it without taking a runtime dependency on `@evf/foundry-mcp`. The 70 entries map 1:1 to the existing `SPELL_LOOKUP` table in `packages/foundry-mcp/src/voice/spell-lookup.ts` and the test asserts that mapping.

Purpose: VOICE-07 (union) + VOICE-08 (locale-aware IT+EN) are land-able on a pure function with full branch coverage. The downstream plans (15-02 wiring, 15-03 refresh, 15-04 failure modes) consume `buildKeytermList()` as a black box.

- `packages/bridge/src/voice/keyterm-merger.test.ts` — KM-01..KM-12 covering union, dedupe, static-wins, locale split, overflow truncation order, empty-cache shape
