<objective>
Ship the Phase 12 LLM-facing contract — the GM-Agent system prompt plus the 3 worked few-shot examples (A: Fireball gruppo / B: dual-wield Action+Bonus / C: clarify ambiguity) — plus the wire-protocol envelope that carries STT transcripts from the bridge through the MCP server. This plan turns CONTEXT.md decisions D-12-02 (system prompt) + D-12-03 (worked examples) into shipping code and locks the `VoiceTranscriptPayloadSchema` that Plan 12-03's Deepgram adapter will emit on the bridge side.

**Purpose:** The GM-Agent system prompt is the contract between the deterministic voice-resolver layer (Plan 12-01) and the Claude Desktop / MCP client that consumes Phase 11's tool surface. Without explicit worked examples, an LLM consuming Phase 11's 6 tool schemas in isolation will hallucinate spell IDs, skip ambiguity checks, and invoke tools without confirming targets. The prompt is the few-shot scaffolding that makes the resolver-table results actually used.

**Output:** Three new files (gm-agent-prompt.ts, worked-examples.ts, payloads/voice.ts) plus the shared-protocol barrel update. Plan 12-03 will consume `VoiceTranscriptPayloadSchema` on the bridge side and the GM-Agent prompt will ship as part of foundry-mcp's published artefact (Phase 11 ResourceCache / docs surface, not a server-injected resource — this plan keeps the prompt as a TS constant that operators paste into Claude Desktop's system-prompt slot; making it a Phase 11-style MCP `resource://` is deferred to V3 per CONTEXT.md deferred-items).

**Research integrated inline:**
- **Phase 11 MCP tool IDs (kebab-case):** Per Phase 11 11-04 SUMMARY decision-4, MCP exposes tool names AS-IS from `TOOL_ID_SCHEMA` which uses kebab-case (`cast-spell`, `weapon-attack`, `use-item`, `move-token`, `place-template`, `drop-concentration`). The bridge layer maps these to snake_case internally (`TO_SNAKE` table in `register-tools.ts`). The GM-Agent prompt MUST use kebab-case — any snake_case slip in the prompt would lead to MCP `tools/list` lookup misses.
- **Phase 11 resource URIs:** `actor://current`, `scene://current`, `combat://current`, `log://recent` (per `EVF_MCP_RESOURCE_URIS` in `register-resources.ts`). The system prompt instructs the agent to read `combat://current` for live target identification (worked example A: gobbi cluster = combatants from combat snapshot).
- **VOICE-05 enforcement:** Per CONTEXT.md domain section + Specs.md §3.1, G2 has no speaker. The prompt explicitly states "no audio output — visual toast only" so that any LLM tempted to generate audio responses is corrected. The substring `visual toast` is grep-asserted to be present.
- **Wire envelope shape:** Mirrors existing Phase 6 `R1_GESTURE_TYPE = 'r1.gesture'` + `R1GesturePayloadSchema` pattern. `R1_VOICE_TRANSCRIPT_TYPE = 'r1.voice.transcript'` keeps the `r1.` prefix even though voice arrives via Hub `audioEvent` (not the R1 ring) — the `r1.` prefix in this codebase means "client-side input event" generally, not literally R1-ring (see `R1_PERF_SAMPLE_TYPE`, `R1_ACTION_RESULT_TYPE`, `R1_MOVEMENT_BUDGET_TYPE` for precedent). VoiceTranscript fields chosen from Deepgram's documented response shape: `transcript` text + `confidence` 0..1 + `is_final` boolean + Deepgram-detected `language` (`it` / `en` / `multi` per nova-3 multilingual; `unknown` for low-confidence detection). `timestamp` is bridge-side `Date.now()` at envelope emission (matches Phase 6 R1GesturePayload precedent).
- **Strict object:** Same pattern as R1GesturePayloadSchema — `z.object(...).strict()` rejects extra fields at the WS-receive trust boundary.

**No DEEPGRAM secrets:** This plan extends `packages/foundry-mcp/src/voice/` (no API key needed — it's the LLM-facing prompt) and `packages/shared-protocol/src/payloads/` (pure Zod, no secrets). Plan 12-01's `voice-no-secret-leak.test.ts` grep gate continues to enforce zero matches on the whole voice subtree.

```typescript
// packages/shared-protocol/src/payloads/voice.ts

export const R1_VOICE_TRANSCRIPT_TYPE = 'r1.voice.transcript' as const;

export const VoiceTranscriptPayloadSchema = z.object({
  /** STT-returned transcript text. Non-empty. */
  transcript: z.string().min(1),
  /** Provider-reported confidence 0..1. */
  confidence: z.number().min(0).max(1),
  /** Deepgram-detected language code (`'multi'` for code-switching). */
  language: z.enum(['it', 'en', 'multi', 'unknown']),
  /** Whether this is a final transcript (vs interim). */
  isFinal: z.boolean(),
  /** Bridge-side Date.now() at envelope emission, ms epoch. */
  timestamp: z.number().int(),
}).strict();

export type VoiceTranscriptPayload = z.infer<typeof VoiceTranscriptPayloadSchema>;
```
