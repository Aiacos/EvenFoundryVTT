<objective>
Ship the deterministic core of the Phase 12 voice path — IT↔EN spell-name fuzzy lookup, Levenshtein distance with accent-insensitive normalisation, and the clarify-detector heuristic that prevents hallucinated spell IDs from reaching the bridge's `tool.invoke` endpoint. This plan delivers VOICE-04 in the resolver layer (external STT integration ships in Plan 12-03; this plan owns the lookup the STT transcript is fed into).

**Purpose:** Without a reliable spell-name resolver, every voice-cast risks an invalid dnd5e canonical ID. Without `clarify-detector`, slang like "scorch 'em" silently maps to an arbitrary spell. Both fail the user. This plan turns CONTEXT.md decisions D-12-01 (lookup table) + D-12-04 (clarify heuristics) into shipping code, fully unit-tested, with zero coupling to the (yet-to-be-built) Deepgram client.

**Output:** A `packages/foundry-mcp/src/voice/` subdirectory with three pure modules (lookup table, Levenshtein, clarify) consumed by Plan 12-02 (GM-Agent prompt + worked examples) and Plan 12-03 (Deepgram WS adapter).

**Research integrated inline:**
- **SRD spell list selection (~70 spells).** The dnd5e 5.3.3 canonical IDs follow kebab-case (`fireball`, `cure-wounds`, `mass-cure-wounds`, `eldritch-blast`). The 70 entries covered here are: all SRD cantrips (~20: acid-splash, chill-touch, dancing-lights, eldritch-blast, fire-bolt, guidance, light, mage-hand, mending, message, minor-illusion, poison-spray, prestidigitation, produce-flame, ray-of-frost, resistance, sacred-flame, shillelagh, shocking-grasp, true-strike, vicious-mockery, thaumaturgy, spare-the-dying), most-used L1 (cure-wounds, healing-word, magic-missile, shield, bless, charm-person, command, detect-magic, disguise-self, faerie-fire, find-familiar, fog-cloud, guiding-bolt, hex, hunters-mark, mage-armor, sanctuary, sleep, thunderwave, witch-bolt, identify, longstrider, jump, feather-fall, burning-hands, chromatic-orb, expeditious-retreat, false-life, color-spray, grease, ray-of-sickness), key L2 (aid, blur, darkness, hold-person, invisibility, misty-step, scorching-ray, see-invisibility, web, spiritual-weapon, lesser-restoration, blindness-deafness, spider-climb, mirror-image, suggestion, knock), key L3 (fireball, counterspell, dispel-magic, fly, haste, lightning-bolt, mass-healing-word, revivify, slow, fear, hypnotic-pattern, vampiric-touch, sleet-storm), and reactions (shield, counterspell, absorb-elements, hellish-rebuke, feather-fall) — the entry list is the authoritative input to the table; tests assert count = 70.
- **Italian translations** follow `dnd5e.it` community localisation (`palla di fuoco` for fireball, `cura ferite` for cure-wounds, `cura ferite di massa` for mass-cure-wounds, `scudo` for shield, `contromagia` for counterspell, `dardo di fuoco` for fire-bolt, `mani magiche` for mage-hand, `raggio di gelo` for ray-of-frost, `dardo incantato` for magic-missile, `parola di cura` for healing-word, `passo nebbioso` for misty-step, `volare` for fly, `velocità` for haste, `tela di ragno` for web, `oscurità` for darkness, `invisibilità` for invisibility, `tenere persone` for hold-person, etc.). Where a community canonical IT name is not unambiguous, the table includes 1-2 common variants (e.g. `cura` AND `cura ferite` both map to `cure-wounds`).
- **Levenshtein normalisation:** apply Unicode NFD then strip combining marks (`\p{M}`), then lowercase, then collapse multi-space — this gives accent-insensitive matching so `velocità`, `velocita`, and `Velocita` all hash to the same canonical key. Code-point iteration via `[...str]` matches the Phase 5 `padRightUnicode` precedent so multi-byte characters cost 1, not 2-3.
- **No DEEPGRAM secrets in this plan.** The Deepgram API key is bound to `packages/bridge/src/voice/deepgram-stt.ts` (Plan 12-03). This plan's grep gate enforces that this subtree has zero references — defence-in-depth against accidental leak.

```typescript
// packages/foundry-mcp/src/voice/spell-lookup.ts

export interface SpellLookupEntry {
  /** dnd5e 5.3.3 canonical ID, kebab-case (e.g. 'fireball', 'mass-cure-wounds'). */
  dnd5eId: string;
  /** Italian display string used for fuzzy lookup (NFD-normalised internally). */
  it: string;
  /** English display string used for fuzzy lookup (NFD-normalised internally). */
  en: string;
  /** Spell level 0..9 (0 = cantrip). */
  level: number;
}

export type SpellLookupConfidence = 'exact' | 'fuzzy' | 'ambiguous' | 'none';

export interface SpellLookupResult {
  /** Canonical dnd5e ID when confidence === 'exact' or 'fuzzy'; null otherwise. */
  dnd5eId: string | null;
  confidence: SpellLookupConfidence;
  /** 'it-table' | 'en-table' | 'levenshtein' | 'no-match' */
  source: 'it-table' | 'en-table' | 'levenshtein' | 'no-match';
  /** Edit distance when confidence === 'fuzzy'; undefined otherwise. */
  distance?: number;
  /** When confidence === 'ambiguous', the tied candidates (dnd5eId + distance). */
  candidates?: Array<{ dnd5eId: string; distance: number }>;
}

export const SPELL_LOOKUP: ReadonlyArray<SpellLookupEntry>;
export function lookupSpellId(transcript: string): SpellLookupResult;
```

```typescript
export type ClarifyReason =
  | 'slang-no-target'
  | 'no-spell-name'
  | 'ambiguous-match'
  | 'empty-transcript';

export interface ClarifyResult {
  needsClarify: boolean;
  reason?: ClarifyReason;
  /** Forwarded from SpellLookupResult when reason === 'ambiguous-match'. */

  resolvedSpellId?: string;
}

export function detectClarify(transcript: string): ClarifyResult;
```
