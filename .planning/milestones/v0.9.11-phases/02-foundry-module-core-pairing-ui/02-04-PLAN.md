<threat_model>
T-02-01 [HIGH]: Bearer token leak in bridge logs — pino logger configured with `redact: ["token", "bearer", "*.token", "*.bearer", "headers.authorization"]` at server creation. Token is never logged at info level; only token hash (first 6 chars for correlation) appears in debug logs behind a `logger.debug()` call that is stripped in production log level (info default).
T-02-02 [HIGH]: Replay attack via seq reuse — `session_store.ts` tracks per-session monotonic seq counter; server rejects any envelope with `seq <= session.lastSeq` as duplicate. On reconnect, `replay_seq` from client is validated against buffer bounds before replaying.
T-02-04 [MEDIUM]: socketlib roundtrip for token validation is a Foundry websocket call — bridge must handle Foundry being unreachable gracefully (return `{ valid: false, reason: "foundry_unreachable" }`) rather than hanging indefinitely. Timeout: 5s on socketlib roundtrip.
T-02-05 [LOW]: 5min cache staleness — documented. The `token-cache.ts` cache-miss path always calls Foundry. Cache keys are token values (never logged); cache evicts on TTL expiry and on explicit `invalidateToken(token)` call exposed for Plan 05 "Refresh now" bridge endpoint.
</threat_model>

**Files:**

- `packages/shared-protocol/src/handshake.ts` — `HandshakeClientSchema` (proto, token, locale, capabilities array), `HandshakeServerSchema` (proto_chosen, server_caps, server_locale, session_id, replay_seq). Export types from both.

- `packages/bridge/package.json` — add `name: "@evf/bridge"`, `dependencies`: fastify 5.8.5, @fastify/websocket 11.2.0, @fastify/cors 11.2.0, @fastify/rate-limit 10.3.0, pino 10.3.1, ws 8.20.0, zod 4.4.3, `@evf/shared-protocol: "workspace:*"`. Dev: vitest 4.1.5, @vitest/coverage-v8, typescript 5.8.3.
- `packages/bridge/src/auth/token-cache.ts` — `TokenCache` class: `validate(token: string): Promise<ValidateTokenResult>`. In-memory `Map<string, { result: ValidateTokenResult; cachedAt: number }>` with 5min TTL (300_000 ms). Cache miss → calls `validateViaFoundry(token)` (async stub returning `{ valid: false, reason: "foundry_unreachable" }` — Plan 04's socketlib roundtrip wires the real call, but the stub allows tests to inject). `invalidateToken(token: string): void` — removes from cache (for "Refresh now" use case).
- `packages/bridge/src/ws/replay-buffer.ts` — `ReplayBuffer` class: per-session FIFO queue of `Envelope` items; max TTL 60s (entries older than 60s evicted on every `push`). `push(env: Envelope): void`; `replay(fromSeq: number): Envelope[]` returns all buffered entries with seq > fromSeq; `size(): number`. Uses `Map<string, Envelope[]>` keyed by session_id.
- `packages/bridge/src/ws/session-store.ts` — `SessionStore` class: `createSession(token: string, locale: string, caps: string[]): Session`; `getSession(sessionId: string): Session | undefined`; `updateLastSeq(sessionId: string, seq: number): void`. `Session` interface: `{ sessionId, token, locale, caps, lastSeq, createdAt }`.

**Action:**
`shared-protocol` Zod schemas must conform strictly to ADR-0002 envelope shape. Do NOT add extra fields — the schema is the contract; adding fields now that Phase 3/4a don't expect causes type drift. `DeltaEnvelopeSchema` is a forward-compatible extension point via `z.unknown()` payload.

`token-cache.ts` constructor accepts an optional `foundryValidateFn: (token: string) => Promise<ValidateTokenResult>` for test injection (no need for mocking libraries — just dependency injection). Production code passes the real socketlib roundtrip function in `server.ts`.

`replay-buffer.ts` eviction: on every `push`, iterate entries for the session and filter out those older than `env.ts - 60_000`. This bounds memory to at most 60s of deltas.

### Task 2: Fastify server + WS handshake + HTTP routes

**Files:**
- `packages/bridge/src/server.ts` — `buildServer(opts?: { foundryValidateFn? }): FastifyInstance`. Registers plugins: `@fastify/cors` (origin from env `EVF_PLUGIN_HOST_URL` or `*` in dev — TODO(#issue) production must pin origin), `@fastify/rate-limit` (100 req/min per IP), `@fastify/websocket`. Sets up pino logger with `redact: ["token", "bearer", "headers.authorization"]`. Registers all routes from `routes/` and WS handler.
- `packages/bridge/src/routes/health.ts` — `GET /v1/health`: validates `Authorization: Bearer {token}` header via `tokenCache.validate(token)`; returns `200 { status: "ok", proto: "evf-v1" }` on valid token; `401 { error: "invalid_token" }` on invalid; `503 { error: "foundry_unreachable" }` when foundry validator returns unreachable.
- `packages/bridge/src/routes/i18n.ts` — `GET /v1/i18n/:lang`: loads `lang/{en,it}.json` from the foundry-module package at server startup (reads from `../../foundry-module/lang/` relative path, resolved at build time). Returns the JSON object. If lang not found, falls back to `en`. No auth required (public endpoint — strings are not sensitive).
- `packages/bridge/src/routes/tools.ts` — `GET /v1/tools`: validates bearer; returns `200 { tools: [] }` (ADR-0003 stub — Phase 7 fills). JSDoc: `/** ADR-0003: Tool registry discovery endpoint. Returns empty array in Phase 2; Phase 7 populates write tools. */`
- `packages/bridge/src/ws/handshake.ts` — `handleHandshake(connection: WebSocket, req: FastifyRequest): Promise<void>`. On WS open: reads first message, validates with `HandshakeClientSchema.safeParse`. If parse fails: close with code 4400, reason "invalid_handshake". Validates token via `tokenCache.validate`. If invalid: close with code 4401, reason "invalid_token". Capability negotiation: `serverCaps = intersection(client.capabilities, SERVER_CAPS_V1)`. Warns (pino.warn) if intersection is smaller than client.capabilities (warn-and-continue per D-2.13). Creates session via `sessionStore.createSession`. Looks up replay buffer for session_id if reconnect (session_id sent in client message for reconnects — first connect has no session_id). Sends `HandshakeServerSchema` response.
- `packages/bridge/src/ws/handshake.test.ts` — tests: valid token → 200 handshake response; invalid token → 4401 close; partial capability overlap → intersection returned + pino.warn called; replay_seq returned from buffer on reconnect.
