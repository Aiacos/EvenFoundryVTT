## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| g2-app/bridge → foundry-module | Untrusted client request crosses into GM-authoritative game state via socketlib.executeAsGM; CI Gate 8 blocks `activity.use()` from leaking back to the client side |
| dispatchTool args (Zod parse) | Untrusted JSON payload validated by handler.argsSchema BEFORE invocation |
| Idempotency cache (bearer-bound) | Replay attempts from different bearers must produce different cache keys (T-03-05 closure) |
| Audit log (whisper: gmIds) | Players cannot read/edit GM-only audit entries; flags.evf.audit immutable on Foundry document |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | g2-app, bridge — client-side `activity.use()` bypass | mitigate | CI Gate 8 grep at compile time; ADR-0011 codifies the constraint; runtime `dispatchTool` is the only entry point from bridge to module write surface |
| T-07-02 | Spoofing | IdempotencyStore — replay with different bearer + same key | mitigate | Cache key = `SHA256(bearer).slice(0,16) + ':' + idempotencyKey` (closes Phase 3 T-03-05 limitation explicitly) |
| T-07-03 | Information Disclosure | audit-log full bearer token leakage | mitigate | `bearer_id` field is `sha256(bearer).slice(0,8)` — NEVER the full token (T-02-01 carry-forward) |
| T-07-04 | Tampering | Audit log entries modifiable by player | accept-with-control | `whisper: gmIds` enforced by Foundry; flags.evf.audit on immutable ChatMessage document; player-side visibility = none |
| T-07-05 | DoS | IdempotencyStore unbounded growth | mitigate | MAX 1000 entries + FIFO eviction (Map insertion order); single-tenant MVP doesn't need bridge's 10_000 ceiling |
| T-07-06 | Repudiation | "I never cast that Fireball" GM dispute | mitigate | Every dispatchTool call writes audit entry with idempotencyKey + actorId + timestamp + bearer_id — GM-side searchable via flags.evf.audit |
