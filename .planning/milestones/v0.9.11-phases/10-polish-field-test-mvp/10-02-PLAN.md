<objective>
Land the latency profiling instrumentation that closes Phase 10 ROADMAP SC-2 in software:
a 5-station perf probe (`gesture_emit → bridge_post → handler_invoke → result_envelope → toast_queued`)
records per-action timestamps, emits an `r1.perf.sample` envelope keyed by a hashed
idempotency key, and ships a `docs/perf/phase-10-latency.md` template that the field-test
operator fills in once real R1+G2+Foundry hardware lands (SC-10-02).

Purpose: without instrumentation we cannot prove the <400ms p50 manual-action latency
target. The probe is opt-in (zero overhead when disabled) so it never affects production
sessions. T-10-02 (D-Area5): the idempotencyKey is hashed via sha256 → first 16 hex chars
so the envelope cannot leak the bearer-bound dedup key, which is sensitive (Phase 3 Plan
01 D-3.07: idempotency keys are tied to a bearer token).

Output: 2 new engine modules (`perf-probe.ts`, `perf-probe-hash.ts`), shared-protocol
envelope schema (`PerfSampleEnvelopeSchema`), boot-engine opt-in wiring, the hardware-pending
docs template.
</objective>
