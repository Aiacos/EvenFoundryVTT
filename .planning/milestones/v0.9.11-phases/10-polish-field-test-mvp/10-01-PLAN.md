<objective>
Land the WS reconnect resilience layer that closes Phase 10 ROADMAP SC-1 in software:
client tracks `lastConfirmedSeq`, surfaces a `⚠ SYNC LOST (riconnetto in Ns)` chip in
the Status HUD footer the moment the WebSocket closes, retries with exponential backoff
`1 → 2 → 4 → 8 → 15 → 30s` (cap 30s), and on reconnect issues a `client_resume {last_seq}`
that the Phase 3 bridge already understands (`packages/bridge/src/ws/resume.ts`
+ `packages/bridge/src/ws/replay-buffer.ts`). Specs §11.5.8.1 verbatim.

Purpose: without this, a bridge restart or 30s network blip silently freezes the HUD on
the player's last-cached state with no recovery signal — the exact failure mode the
multi-session field test (SC-10-01) is designed to surface. The chip is also a Mode-Driver
for INV-5 (visible enforcement: gestures route to the same top layer but write paths are
implicitly degraded — the user must see it).

Output: 3 new engine modules (`seq-tracker.ts`, `ws-reconnect.ts`, `sync-lost-chip.ts`),
status-hud-renderer integration, boot-engine wiring, 2 INV-1 fixtures (IT + EN), per-D-Area1
threat T-10-01 mitigation (gap detection → buffer_expired → full refresh).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ws → client (inbound) | Bridge-originated envelopes arrive over WS; client trusts only the parsed envelope shape, never the seq numbers blindly for replay decisions |
| client → bridge (outbound) | `client_resume` is bearer-authenticated; the controller MUST NOT include any user-controlled string fields beyond the planned schema (session_id is captured from the original handshake, never re-derived) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering / Information disclosure | `WsReconnectController.handleResumeResponse` (stale last_seq could request envelopes older than the buffer, causing a partial replay that hides a real game-state mutation) | mitigate | Honor `resume_full_snapshot { reason: 'buffer_gap' | 'buffer_expired' }` by calling `seqTracker.reset()` + `onFullRefreshRequired()` BEFORE forwarding any further envelopes. Server-side gap detection in `packages/bridge/src/ws/resume.ts` is the authority; client never bypasses it. Verified in test WSR-07 |
| T-10-02 | Denial of service | `WsReconnectController` backoff loop (infinite retry if bridge is permanently down) | accept | Backoff caps at 30s — single-tenant homelab MVP. Long-disconnect UX is a SYNC LOST chip with countdown; user can manually quit the Even Realities App. Multi-tenant DoS is Phase 13 stretch concern |
</threat_model>
