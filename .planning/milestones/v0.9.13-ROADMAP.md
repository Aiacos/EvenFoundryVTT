## Milestone Summary

**Key Decisions:**

- Atomic schema-extension pattern reaffirmed for Phase 16 + Phase 17 (Phase 4b/5 precedent): schema REQUIRED with no `.optional()` window; downstream consumer test snapshots fixed atomically per wave.
- `proficient` semantics differ between abilities (boolean coerced) and skills (verbatim 0|0.5|1|2 enum): Main tab is binary, Skills tab needs the full spectrum for ○/◉/★ glyph mapping.
- Half-prof `0.5` rounds UP to `◉` glyph (rationale: half-prof still adds proficiency bonus; treating as "proficient-ish" preserves data honesty within INV-1 width budget).
- Static `SKILL_NAMES` / `PASSIVE_ABBR` const tables in renderer-side TS (no Foundry runtime available plugin-side).
- Option (a) doc-fix preferred over fixture re-padding for §10 width-budget drift (lower-risk doc-only INV-3 atomic commit).
- Single INV-3 atomic commit per phase close, per Phase 14 `3a0c5cf` / Phase 15 `dc161d6` precedent.
- Phase 18 bundles WR-UI polish with milestone close (Specs.md v0.9.13 bump + README + showcase) in single INV-3 atomic, then audit → complete → cleanup sequence.

**Issues Resolved:**

- 9 v1 REQ-IDs closed (SHEET-05..10 + INFILL-14.1-A..C).
- 3 advisory UI-REVIEW carry-forward findings from Phase 14 closed (WR-UI-01/02/03).
- 4 additional IT-locale fixture leaks discovered + fixed beyond original UI-REVIEW enumeration (rows 5/7/9/12).
- Z05-INV-02b regression-detection gap closed via triade extension.
- 23 downstream snapshot literals across 17 test files extended for `abilities` + `skills` fields.

**Issues Deferred:**

- Inventory/Bio/Feats tab data-binding polish — outside this milestone's scope.

- Hardware UAT closure — 35 SCs from v0.9.11 carry under ADR-0005 PROVISIONAL Branch A unchanged (no Even Hub access this cycle).

**Technical Debt Incurred:**
