<objective>
Build `CanvasCharacterSheetPanel` — the first z=2 canvas overlay panel — as a dual-interface
class implementing BOTH `CanvasLayer` (canvas compositor path) AND `OverlayPanel` (panel-router
lifecycle + gesture bus). Add additive `paint*Tab(ctx, ...)` canvas renderers alongside the
PRESERVED `render*Tab()` string renderers, wire gesture-identical tab navigation, and gate
boot-time panel dispatch on `renderMode` so the canvas panel opens in canvas mode and the glyph
panel opens in glyph mode.

This plan deliberately EXCLUDES the portrait pipeline (Plan 21-04, which touches the same panel
file in a later wave). It delivers the painting + tab + gesture substrate with a stubbed/absent
portrait.

Purpose: RSHEET-01 (6-tab canvas sheet, chrome pre-bake, dirty-gate) + RSHEET-02 (gesture-identical
navigation, panel-gesture-bus unchanged). Main tab consumes the class/initiative/speed fields from
Plan 21-01.
Output: dual-interface panel + paint*Tab methods + boot dispatch + RCSP-SC tests.
</objective>
