<threat_model>
T-02-01 [HIGH]: Delta envelopes must never include the raw bearer token. `hook-subscribers.ts` sends a POST to bridge `/internal/delta` with a server-to-server shared secret (env `EVF_INTERNAL_SECRET`, 32-byte random set at bridge startup, exchanged via Foundry module settings at pair time). This is separate from the client bearer. The internal secret is redacted from pino logs.
T-02-02 [HIGH]: seq monotonicity — `delta-emitter.ts` maintains a per-session atomic counter. Each emitted envelope gets `seq = ++sessionSeq`. Bridge rejects any client ack with out-of-order seq.
T-02-04 [MEDIUM]: socketlib handlers for snapshot reads validate token before executing Foundry query. `evf.getCharacterSnapshot(actorId, token)` — handler calls `validateBearer(token)` first; returns null if invalid.
T-02-05 [LOW]: ring buffer Foundry-side (200 entries) vs replay buffer bridge-side (60s LRU) serve different purposes. Ring buffer is for event-log cursor pagination (REST `?since=N`). Replay buffer is for WS reconnect. These must not be confused. Ring buffer index is a monotonic integer cursor; replay buffer TTL is wall-clock.
</threat_model>

<objective>
Implement the Foundry-side reader API: 5 hook subscribers (D-2.14), 200-entry ring buffer for event log (D-2.16), and socketlib GM handlers for snapshot reads. On the bridge side: 5 REST snapshot routes + `/v1/characters` list endpoint + WS delta emitter that fans out to all subscribed sessions and pushes to replay buffer. Define Zod payload schemas in `@evf/shared-protocol` for all 4 resource types.

Purpose: This is the one-way read pipeline goal of Phase 2. Without it, FOUN-01 and CONN-01 are unmet — the phone wizard completes but the G2 display (Phase 4a) has no data to render.

## Task Breakdown

### Task 1: Foundry-side readers + ring buffer + hook subscribers

**Files:**
- `packages/foundry-module/src/readers/ring-buffer.ts` — `class RingBuffer<T>` with capacity 200. `push(item: T): void` — overwrites oldest on overflow. `toArray(): T[]` — returns items in insertion order (oldest first). `since(seq: number): T[]` — returns items with `item.seq > seq`. Internal: fixed-length `T[]` array + head/tail pointers.
- `packages/foundry-module/src/readers/character-reader.ts` — `getCharacterSnapshot(actorId: string): CharacterSnapshot | null`. Reads from `game.actors.get(actorId)`. Validates actor exists and is of type "character". Extracts: hp, maxHp, tempHp, ac, level, name, conditions (from `actor.statuses`), exhaustion, actorId. Returns null if actor not found or not PC. Uses `noUncheckedIndexedAccess`-safe property access (explicit undefined checks per D-1.04).
- `packages/foundry-module/src/readers/combat-reader.ts` — `getCombatSnapshot(): CombatSnapshot | null`. Reads `game.combat` (may be null if no active combat). Returns null → bridge route returns 204. Extracts: combatId, round, turn, current combatant, all combatants with hp/initiative.
- `packages/foundry-module/src/readers/scene-reader.ts` — `getSceneViewport(): SceneViewport`. Reads `game.scenes.active` + `canvas.stage` position. Returns sceneId, sceneName, viewX, viewY, scale.
- `packages/foundry-module/src/readers/event-log-reader.ts` — `getEventLog(since: number, limit: number): EventLogEntry[]`. Reads from the `RingBuffer<EventLogEntry>` module-level instance. Returns `buffer.since(since).slice(0, limit)`.
- `packages/foundry-module/src/readers/hook-subscribers.ts` — `registerHookSubscribers(emitFn: (type: string, payload: unknown) => void): () => void`. Registers 5 hooks per D-2.14. Returns cleanup function that calls `Hooks.off(id)` for all registered hooks. Each hook: reads snapshot via reader functions, calls `emitFn(type, snapshot)`. For `updateActor`: only emit if `changes.system?.attributes?.hp || changes.system?.attributes?.ac || changes.statuses` (avoid spurious emits on unrelated actor changes).
- `packages/foundry-module/src/module.ts` — update `Hooks.once("ready")` to call `registerHookSubscribers(bridgeDeltaEmitter)`. `bridgeDeltaEmitter` is a function that POSTs to bridge `/internal/delta` with internal secret.
- `packages/foundry-module/src/readers/readers.test.ts` — mock `game.actors`, `game.combat`, `game.scenes`, `canvas`. Test: getCharacterSnapshot returns correct shape for mock actor; getCombatSnapshot returns null when game.combat is null; getSceneViewport returns correct sceneId; updateActor hook emits only on HP/AC/status changes.
- `packages/foundry-module/src/readers/ring-buffer.test.ts` — test: push 201 items → oldest evicted; since(N) returns correct subset; toArray returns in order.
- `packages/foundry-module/src/types/foundry-globals.d.ts` — expand with hook signatures, `game.actors`, `game.combat`, `game.scenes`, `game.user.targets`, `canvas`.

**Action:**
`registerHookSubscribers` accepts `emitFn` as a parameter (dependency injection) — makes it fully testable without a real bridge. The `bridgeDeltaEmitter` passed from `module.ts` is a fire-and-forget async function that POSTs to bridge. It logs on failure (biome-ignore warn line) but does not throw — a failed bridge emit must not crash the Foundry session.

The `updateActor` guard is critical for performance (D-2.15 zero polling): only emit when the relevant fields change. The `changes` parameter in the hook is a partial diff object — check `changes.system?.attributes` exists before reading deeper (noUncheckedIndexedAccess compliance).

For FOUN-04 (`targetToken` hook): emit `combat.targets` payload with `{ userId, targets: Array<{ tokenId, actorId, name }> }` where targets are read from the hook parameters (token + targeted flag). Phase 7 will add the setTargets *write* call; Phase 2 is read-only observation.

### Task 2: Bridge REST routes + WS delta emitter + shared-protocol payloads

**Files:**
- `packages/shared-protocol/src/payloads/character.ts`, `combat.ts`, `scene.ts`, `event.ts` — Zod schemas as specified in interfaces. Export types alongside schemas.
- `packages/shared-protocol/src/index.ts` — add re-exports for all payload schemas.
- `packages/bridge/src/ws/delta-emitter.ts` — `class DeltaEmitter`. Maintains `Map<sessionId, WebSocket>` of live connections. `registerSession(sessionId, ws)` + `unregisterSession(sessionId)`. `emitDelta(type: string, payload: unknown): void`: creates `Envelope { proto: "evf-v1", seq: ++globalSeq, ts: Date.now(), type, session_id: "*", payload }`. For each registered session: validates session has matching capability (e.g. `read_char` for `character.delta`), serializes + sends. Also calls `replayBuffer.push(envelope)` for each session.
- `packages/bridge/src/routes/character.ts` — `GET /v1/character/:actorId`. Auth via bearer. Calls `socketlib.executeAsGM("evf.getCharacterSnapshot", actorId, token)` (roundtrip to Foundry). Returns 200 + `CharacterSnapshotSchema` JSON or 404 if null returned.
- `packages/bridge/src/routes/combat.ts` — `GET /v1/combat/current`. Returns 200 + `CombatSnapshotSchema` or 204 (no content) if null.
- `packages/bridge/src/routes/scene.ts` — `GET /v1/scene/viewport`. Returns 200 + `SceneViewportSchema`.
- `packages/bridge/src/routes/events.ts` — `GET /v1/events?since=N&limit=200`. Calls `socketlib.executeAsGM("evf.getEventLog", since, limit, token)`. Returns `{ entries: EventLogEntry[], cursor: number }`.
- `packages/bridge/src/routes/characters-list.ts` — `GET /v1/characters?world=`. Calls socketlib `evf.listCharacters(worldId, token)` → returns array of `{ actorId, name, class, level }`. Required by wizard Step 3.
- `packages/bridge/src/routes/internal-delta.ts` — `POST /internal/delta`. Auth via `EVF_INTERNAL_SECRET` header (not bearer — this is module → bridge internal). Validates secret, validates body against `DeltaEnvelopeSchema`, calls `deltaEmitter.emitDelta(type, payload)`.
- `packages/foundry-module/src/readers/hook-subscribers.ts` — add socketlib handlers for new endpoints: `evf.getCharacterSnapshot`, `evf.getCombatSnapshot`, `evf.getSceneViewport`, `evf.getEventLog`, `evf.listCharacters`. Each handler: validate token, execute reader, return result.
- `packages/bridge/src/ws/delta-emitter.test.ts` — test: emitDelta fans out only to sessions with matching cap; replayBuffer.push called once per envelope; globalSeq increments monotonically.
- `.changeset/02-05-reader-api.md` — minor bump for `@evf/foundry-module`, `@evf/bridge`, `@evf/shared-protocol`.

**Action:**
`DeltaEmitter` global seq is per-bridge-instance (not per-session). Each envelope gets a unique global seq. Sessions track `lastSeq` via `session-store.ts`. On reconnect, the WS handshake returns the last globally-known seq in `replay_seq`; client requests `replay(fromSeq)` from `ReplayBuffer`.

The `/internal/delta` route uses a separate Fastify plugin registration: it binds only on `127.0.0.1` (localhost-only) in production — the Foundry module and bridge run on the same Docker Compose network. Add a note: `// TODO (#43): restrict /internal/delta to Docker internal network in production`.

socketlib handlers added here (`evf.getCharacterSnapshot` etc.) are registered in `registerSocketlibHandlers()` from Plan 02 — update that function to include the new handlers. Do NOT duplicate handler registration in `registerHookSubscribers` — keep concerns separated: hook-subscribers only call emitFn; socketlib-handlers only serve snapshot reads.

All Zod payload schemas must be `z.strictObject(...)` — fail on unknown fields (prevents schema drift silently accepting new Foundry fields we didn't plan for).
