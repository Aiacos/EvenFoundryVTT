<threat_model>
T-02-01 [HIGH]: Bearer token leak in logs — generateBearer must NEVER log the token value. pino redact list scaffolded in bearer-registry.ts. Token stored in Foundry settings (Tier 3) uses a setting key `evf.bearerRegistry` of scope `world` (DM-authoritative). The raw token value is never included in ApplicationV2 rendered HTML — only alias + expiry date are rendered.
T-02-02 [HIGH]: Replay attack on WS — bearer-registry.ts enforces token TTL at validation time (Date.now() vs expires). Grace period on rotation: old token valid for 60s after `generateBearer(refresh=true)` to avoid race with in-flight requests.
T-02-04 [MEDIUM]: socketlib handler accepts unauthenticated calls — `evf.validateToken(token)` validates the token itself before reading any world state. The handler cannot be called as GM without a valid socketlib session, but double-check: handler must return `{ valid: false, reason: "unknown_token" }` for any unregistered token rather than throwing.
T-02-05 [LOW]: Revoke registry staleness — explicitly documented; "Refresh now" button on the bridge side (Plan 04) forces cache invalidation. Foundry-side revoke is immediate.
</threat_model>

### Task 1: Bearer registry

**Files:**
- `packages/foundry-module/src/pair/bearer-registry.ts`
- `packages/foundry-module/src/pair/bearer-registry.test.ts`
- `packages/foundry-module/package.json` — add `qrcode@1.5.4` to `dependencies` (needed for PairModal in Task 2)

**Action:**
Implement `generateBearer(alias: string, bridgeUrl: string, worldId: string, refresh?: boolean): Promise<BearerEntry>`:
- Use `crypto.getRandomValues(new Uint8Array(32))` → base64url encode (same as `foundry.utils.randomID` but explicit for testability)
- Store in `game.settings.get("evenfoundryvtt", "bearerRegistry")` as `BearerRegistry` (Tier 3 DM setting, `scope: "world"`, `config: false` — hidden from UI, managed programmatically)
- If `refresh=true`: new entry created, old entry gets `expiresAt = Date.now() + 60_000` (60s grace period per D-2.11)

Implement `validateBearer(token: string): { valid: boolean; entry?: BearerEntry; reason?: string }`:
- Looks up token in registry; returns `{ valid: false, reason: "unknown_token" }` if not found
- Returns `{ valid: false, reason: "revoked" }` if `revokedAt !== null`
- Returns `{ valid: false, reason: "expired" }` if `expiresAt < Date.now()`
- Returns `{ valid: true, entry }` otherwise

Implement `revokeBearer(token: string): void`:
- Sets `revokedAt = Date.now()` in registry; persists via `game.settings.set`

Implement `listBearers(): BearerEntry[]`:
- Returns all non-revoked entries sorted by `createdAt` descending

In `bearer-registry.test.ts`: mock `game.settings` (get/set). Test: generateBearer produces 32+ char base64url token; validateBearer returns false for unknown/revoked/expired; revokeBearer sets revokedAt; silent refresh creates new entry. Use `tdd="true"` cycle: write failing tests first.
