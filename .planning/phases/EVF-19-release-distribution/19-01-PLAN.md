<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| workflow_dispatch input `tag` → workflow | Operator-supplied tag string crosses into shell + image tagging |
| GitHub Actions runner → GHCR | CI pushes a container image to the public registry |
| CI build → GitHub Release assets | CI uploads installable artifacts end users will trust |
| third-party `docker/*` actions → CI runner | external action code executes in the release pipeline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Tampering | workflow_dispatch `tag` input | mitigate | Existing tag regex gate `^[0-9]+\.[0-9]+\.[0-9]+(-[A-Za-z0-9.-]+)?$` in the `vars` step is reused; new jobs derive version from the same validated output (no second unvalidated parse) |
| T-19-02 | Elevation of Privilege | GITHUB_TOKEN scope | mitigate | Per-job permissions: `packages: write` granted ONLY to build-bridge-ghcr; release/g2app jobs get `contents: write`; top-level grant removed (least privilege, ASVS V4) |
| T-19-03 | Spoofing | GHCR authentication | mitigate | docker/login-action with `secrets.GITHUB_TOKEN` (auto-scoped, short-lived) — NO PAT introduced; username is `github.actor` |
| T-19-04 | Information Disclosure | secret leakage in logs / image | accept-with-control | deploy/bridge.Dockerfile already takes no secret build-args (T-03-17); no `echo`/`cat` of secrets added; GITHUB_TOKEN masked by Actions automatically |
| T-19-05 | Tampering | release-asset integrity | mitigate | Assets produced only from checked-out source in CI (no external download into the zip); `--clobber` re-uploads are deterministic from the tagged source tree |
| T-19-SC | Tampering | third-party docker/* actions (supply-chain) | mitigate | All `docker/*` + `actions/*` pinned to VERIFIED majors from RESEARCH.md; no `@main`/floating refs; no new npm/pip/cargo installs in this phase (CI Gate 8 socketlib=17 untouched) |
</threat_model>
