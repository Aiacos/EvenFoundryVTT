| bridge → g2-app | WS envelopes cross the only untrusted hop into the panel layer — double safeParse mandatory. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | `combat-action-tracker.ts` filter logic | mitigate | Filter strictly on `flags.evf.audit.toolId in {cast-spell, weapon-attack, use-item}`. Audit entries from `drop-concentration`, `move-token`, `place-template`, `confirm-template-placement` MUST NOT increment counters (CAT-05 test). |
| T-09-02 | Spoofing | multi-attack `attackId` collision across players | mitigate | `_attackIdSeen` keyed by `(actorId, attackId)` tuple — two players who happen to share an attackId (defensive, never expected since Phase 7 uses `crypto.randomUUID()`) are still scoped to their own actor partition. CAT-03 verifies dedupe; CAT-10 verifies actorId scoping. |
| T-09-03 | Information disclosure | cross-player action economy leak | mitigate | `payload.recipientUserId !== currentUserId` → SILENT drop in dispatcher (AED-05). No warn — prevents attacker from confirming envelope receipt (T-08-02 pattern from action-result-dispatcher.ts). |
| T-09-04 | Repudiation | audit entry missing attackId for multi-attack chat cards | mitigate | Plan 09-01 Task 1 extends `audit-log.ts` to write `flags.evf.audit.attackId` when the handler result carries it. AL-EXT-01 verifies the propagation. |
| T-09-05 | Denial of service | Hook handler throws crashing Foundry session | mitigate | Defensive try/catch with `console.warn` in BOTH hook bodies. CAT-09 verifies. Pattern inherited from `combat-movement-tracker.ts` and `action-result-watcher.ts`. |
