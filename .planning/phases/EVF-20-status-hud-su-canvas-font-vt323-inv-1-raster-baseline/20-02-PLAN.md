<objective>
Establish the raster INV-1 contract: a deterministic synthetic RGBA drawn from a
fixed known Status-HUD state, run through `buildHudTiles()`, with each of the 4
resulting PNG tiles SHA-256-hashed and compared against a committed golden
fixture. This is the raster-suite counterpart to the existing glyph ASCII
fixtures (RINV-01).

Purpose: INV-1 layout integrity must extend to the raster substrate. Canvas-text
output is non-deterministic in happy-dom (locked decision RINV-01 forbids hashing
it), so the contract hashes the deterministic `buildHudTiles()` pipeline output
from a synthetic RGBA — reproducible byte-for-byte from code, no binary blobs.
Output: `20-raster-inv1.test.ts` (RINV-01) + the committed
`status-hud.raster-hash.json` golden fixture.
</objective>
