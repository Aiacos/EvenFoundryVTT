<objective>
Implement `CanvasStatusHudLayer` — the first real `CanvasLayer` (z=1) — that
pre-bakes the static HUD chrome once into an `ImageBitmap` cache at mount
(RFONT-02), GPU-blits it under dynamic data on each paint, re-renders only its own
layer on `character.delta` reusing the `hud-live-render.ts` snapshot pattern
(RFONT-03), and paints only when `isDirty()` (SC3) so idle frames cost nothing.

Purpose: This is the canvas replacement for the glyph `StatusHudLayer`. It proves
the Phase 19 substrate (CanvasCompositor + async attachCanvas + CanvasLayer
contract) renders a real layer, and it is BLE-safe ahead of the Phase 24 delta
loop because the dirty-gate guarantees near-zero idle bandwidth.
Output: `CanvasStatusHudLayer` + SC2/SC3 unit tests, all green in happy-dom.
</objective>
