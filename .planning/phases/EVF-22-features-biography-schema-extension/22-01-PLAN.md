<objective>
Add the wire-contract for Phase 22: two new Zod schemas (`FeatEntrySchema`, `BiographySnapshotSchema`) in `@evf/shared-protocol` and wire them as OPTIONAL fields onto `CharacterSnapshotSchema`. This is the single source of truth all downstream packages (foundry-module reader, g2-app renderers) consume.

Purpose: RDATA-03 / RDATA-04 require `feats[]` and `biography` on the snapshot. Making them OPTIONAL (per D-22.1) means the ~26 downstream `CharacterSnapshot` literals across g2-app/bridge/foundry-mcp suites compile under TS strict with NO mass update — exactly the pain avoided from Phase 21-01's REQUIRED `class` field.
Output: extended `character.ts` schema + new exported types + `CS-FE-*` / `CS-BIO-*` tests in `character.test.ts`.
</objective>
