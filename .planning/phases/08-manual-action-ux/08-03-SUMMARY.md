## What Was Built

### ActionOptionsModal (`action-options-modal.ts`)

Z=2 OverlayPanel following Strategy A (single `overlay-block` text container). Constructor: `(bridge, ws, gestureBus, request: ActionOptionsRequest, locale, sessionId, onClose)`.

Key behavior:
- **Tap**: if `requiresTarget === false` → emits canonical `EnvelopeSchema` `tool.invoke` envelope (`toolId: 'cast-spell'` or `'use-item'`, arg key `spell_id` / `item_id`) then calls `onClose()`. If `requiresTarget === true` → calls `onClose()` without emit (Plan 08-05 will intercept and push TargetPickerPanel).
- **Long-press**: console.warn stub (Phase 9 detail viewer).
- **Double-tap**: `onClose()` without emit (cancel).

Frame layout (60×12, `┌─[ AZIONE ]─...─┐` border):
```
┌─[ AZIONE ]───────────────────────────────────────────────┐
│                                                          │
│   AZIONE: Palla di Fuoco                                 │
│                                                          │
│   [tap]   Lancia incantesimo                             │
│   [long]  Mostra dettagli                                │
│   [×2]    Annulla                                        │
│                                                          │
...
└──────────────────────────────────────────────────────────┘
```

### SpellbookPanel long-press wiring

Added `setActionOptionsHandler(handler)` setter. When set, `onEvent('long-press')` constructs an `ActionOptionsRequest` from the currently highlighted spell at `scrollOffset`:
- `kind: 'spell'`, `actorId: snapshot.actorId`, `itemId: spell.id`
- `requiresTarget = spell.range !== 'self' && spell.range !== '' && spell.activation !== 'reaction'`

When handler is null (default), long-press remains a no-op — Phase 6 router-level dispatcher still fires (backward-compat).

### InventoryPanel long-press wiring

Mirror pattern. `requiresTarget = item.type !== 'consumable'` (consumables self-target by default). Dispatches `kind: 'item'`.

### Design Decision: setter vs constructor arg

Plan original text suggested adding `openActionOptions` as a constructor argument. Chosen implementation uses `setActionOptionsHandler(handler)` setter instead. Rationale: `PanelRouter.discoverPanels` uses a `PanelConstructor` type with exactly 3 args `(bridge, gestureBus, locale)`. Adding a 4th constructor arg would break all `PanelRouter`-managed instantiation paths without surgical updates to PanelConstructor. The setter pattern allows Plan 08-05 to wire the handler after mount without touching PanelRouter.
