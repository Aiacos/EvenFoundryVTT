<objective>
Build the canvas compositor substrate (RAST-01, RAST-02): the `CanvasCompositor` class (master 400×200 canvas + z-order `drawImage` composite + dirty-skip), the additive `CanvasLayer` interface (per-layer offscreen canvas ownership), and the `buildHudRasterPageSchema()` 5-container page schema (4 image tiles + 1 full-screen text capture). No `CanvasLayer` implementations are added in Phase 19 — the interface is defined here and Phase 20 provides the first concrete layer. `CanvasCompositor.composite()` returns a blank 400×200 RGBA buffer when no layers are registered.

Purpose: This is the reusable substrate every subsequent canvas-rendering phase builds on. It is wired into `LayerManager` in plan 19-04. Implemented per ADR-0013 Amendment 1 (ratified in plan 19-01) and the 400×200/200×100 geometry (migrated in plan 19-02).

Output: New `canvas-compositor.ts` + test; `CanvasLayer` additions to `layer-types.ts`; `buildHudRasterPageSchema()` + HUD container registry entries + test additions.
</objective>
