<objective>
Expose the Phase 2 reader pipeline as MCP resources: actor/scene/combat snapshots + a 50-entry log ring. Each resource is read-through-cache: the cache is primed by WS delta subscriptions from the bridge, and a REST fallback handles cold starts and reconnects. After this plan, an MCP client (Claude Desktop / Inspector) can `resources/read actor://current` to fetch the latest CharacterSnapshot without polling, and `resources/subscribe` to receive change notifications via `sendResourceUpdated`.

**Purpose:** Delivers VOICE-03 ("resources are exposed"). Enables an LLM to ground its tool calls in current world state (e.g., "the actor has 12 HP, currently concentrating on Bless") without re-fetching on every turn.

**Output:** ResourceCache + WS subscription + 4 registered resources with sendResourceUpdated notifications.
</objective>

    import type { CharacterSnapshot, CombatSnapshot, SceneViewport, EventLogEntry } from '@evf/shared-protocol';

    type ResourceUri = 'actor://current' | 'combat://current' | 'scene://current' | 'log://recent';
    type ResourceValueOf<U extends ResourceUri> =
      U extends 'actor://current' ? CharacterSnapshot :
      U extends 'combat://current' ? CombatSnapshot :
      U extends 'scene://current' ? SceneViewport :
      U extends 'log://recent' ? EventLogEntry[] :
      never;

    /**
     * Internal FIFO ring buffer for log entries. Local-only — Phase 11 does not
     * promote the Phase 2 foundry-module ring-buffer to shared-protocol.
     */
    class LogRing {
      private buf: EventLogEntry[] = [];
      constructor(private capacity: number) {}
      push(entry: EventLogEntry): void {
        this.buf.push(entry);
        if (this.buf.length > this.capacity) this.buf.shift();
      }
      toArray(): EventLogEntry[] { return [...this.buf]; }
      clear(): void { this.buf = []; }
    }

    export class ResourceCache {
      private store = new Map<ResourceUri, unknown>();
      private subscribers = new Map<ResourceUri, Set<() => void>>();
      private logRing = new LogRing(50);
      get<U extends ResourceUri>(uri: U): ResourceValueOf<U> | undefined { ... }
      set<U extends Exclude<ResourceUri, 'log://recent'>>(uri: U, value: ResourceValueOf<U>): void { ... fire subscribers }
      appendLog(entry: EventLogEntry): void { ... fire log://recent subscribers }
      onUpdate(uri: ResourceUri, cb: () => void): () => void { ... return unsubscribe }
      clear(): void { ... }
    }
    ```

    export const EVF_MCP_RESOURCE_URIS = ['actor://current','combat://current','scene://current','log://recent'] as const;
    export type EvfMcpResourceUri = typeof EVF_MCP_RESOURCE_URIS[number];

    const RESOURCE_META: Record<EvfMcpResourceUri, { name: string; title: string; description: string }> = {
      'actor://current':  { name: 'actor-current',  title: 'Current Actor',  description: 'Latest CharacterSnapshot of the player-controlled actor — HP, AC, conditions, level, death saves, modernRules, inventory, spellbook. Updated live via WS character.delta envelopes from FoundryVTT.' },
      'combat://current': { name: 'combat-current', title: 'Current Combat', description: 'Active combat tracker state — round, turn, initiative order, current combatant, concentration. Null when no combat is active.' },
      'scene://current':  { name: 'scene-current',  title: 'Current Scene',  description: 'Scene viewport — dimensions, grid, current camera position. Source for token placement coordinates passed to move-token.' },
      'log://recent':     { name: 'log-recent',     title: 'Recent Event Log', description: 'Last 50 event-log entries (attack, damage, spell, feature, round, concentration, roll, chat). Use to ground tool calls in recent narrative context.' },
    };

    export function registerEvfResources(server: McpServer, cache: ResourceCache, bridgeClient: BridgeClient, logger: Logger): void {
      for (const uri of EVF_MCP_RESOURCE_URIS) {
        const meta = RESOURCE_META[uri];
        server.registerResource(meta.name, uri, { title: meta.title, description: meta.description, mimeType: 'application/json' }, async (url) => {
          const value = await readResource(uri, cache, bridgeClient, logger);
          return { contents: [{ uri: url.toString(), mimeType: 'application/json', text: JSON.stringify(value) }] };
        });

        // Wire sendResourceUpdated to cache changes
        cache.onUpdate(uri, () => {
          server.sendResourceUpdated({ uri }).catch((err) => logger.warn({ err, uri }, 'sendResourceUpdated failed'));
        });
      }
    }

    async function readResource(uri: EvfMcpResourceUri, cache, bridgeClient, logger): Promise<unknown> {
      const cached = cache.get(uri);
      if (cached !== undefined) return cached;
      // REST fallback
      switch (uri) {
        case 'actor://current': return await bridgeClient.getCharacterSnapshot(/* MVP: actorId from env if present */);
        case 'combat://current': return await bridgeClient.getCombatSnapshot();
        case 'scene://current': return await bridgeClient.getSceneViewport();
        case 'log://recent': return await bridgeClient.getEventLog(50);
      }
    }
    ```

| Bridge WS delta → ws-subscription handler | Untrusted payload; Zod parse via Phase 2 schemas validates before cache write |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-11 | Tampering | Bridge-supplied delta payload | mitigate | Per-type Zod safeParse in ws-subscription; failed parse → drop + warn log; no cache mutation. Phase 2 schemas already validated by foundry-module readers; this is defense-in-depth. |
| T-11-12 | Information disclosure | Resource payloads to LLM | accept | LLM has the same view as the paired player would on G2. CharacterSnapshot is the player's own actor (not other players'). Multi-player isolation is Phase 13 stretch. |
| T-11-13 | Denial of service | Delta floods filling LogRing | mitigate | LogRing capacity=50; oldest entries dropped FIFO. Per-resource single-snapshot (no growth). Bounded memory. |
| T-11-14 | Repudiation | LLM acts on stale data | accept | Cache freshness is best-effort; bridge can disconnect/reconnect. REST fallback re-fetches; LLM responsible for cross-checking via subsequent resource reads when uncertain. |
