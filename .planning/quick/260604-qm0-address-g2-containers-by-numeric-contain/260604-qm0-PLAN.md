<objective>
Make the G2 HUD actually render on the glasses. The EvenHub host requires every container to be addressed by its numeric `containerID` (it rejects `container_id is required` when only `containerName` is sent), and text containers currently have NO geometry (so even a correct id renders them at 0,0 size 0 -> invisible). Both gaps are root-caused and probe-validated in `.planning/debug/glasses-render-blank-containerid.md`.

This plan introduces ONE shared container registry (name -> {id, geometry, isEventCapture}), assigns id+geometry in both page schemas, repairs `LayerManager._flushPage` (which currently rebuilds an EMPTY page and wipes all containers after boot), threads the numeric `containerID` into every `textContainerUpgrade` / `updateImageRawData` call site, updates the affected tests, and PROVES the fix in the EvenHub simulator (no container errors + non-blank glasses screenshot).

Purpose: this is the last blocker between "engine boots + WS connects + handshake passes" and "the player sees the HUD on the glasses."
Output: a registry module, repaired page-schema + flush path, id-threaded call sites, a registry unit test, a bumped @evf/g2-app version + changeset, and a captured non-blank glasses screenshot as runtime evidence.
</objective>
