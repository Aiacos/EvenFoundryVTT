<objective>
Redesign the DEFAULT always-on glasses HUD to the real G2 27px font grid. The G2 LVGL
font has a FIXED 27px line height (no font control per SDK): 576×288 → ~10 rows max;
a full-width 576px line ≈ ~50 chars (variable-width). The current HUD was built for a
~12px/24-row grid (status-hud-renderer = 28-char × 21-row card; container-registry
geometry h12/h24), so on real glasses the text is ~2.25× too big and overlaps/clips
("scritte troppo grandi", user 2026-06-05).

This plan replaces the DEFAULT always-on view ONLY with a full-width character status
sheet (~50 chars × ~9 rows) per the user-approved mockup, makes that sheet the boot
base layer (the raster map is NOT the default base anymore), and updates the INV-1
fixtures + Specs/ADR/README/showcase atomically (INV-3).

Approved mockup (the design contract):
  Dante Lanzulli            Lv10 Chierico
  ----------------------------------------
  HP #######... 41/63   CA 16   VEL 9m
  Turno 2/5   Round 3   [TUO TURNO]
  Cond: concentrato, benedetto
  ----------------------------------------
  Slot 1o***o  2o**o  3o*o
  TS morte  ooo / ooo
  R1: ^v scorri  tap ping  oo menu

Purpose: fix the only known shipping bug blocking real-glasses readability.
Output: a clean, correctly-sized full-width status sheet on the default view, verified
live via `pnpm sim start`.

## SCOPE (bounded — read carefully)

IN SCOPE: the DEFAULT always-on status view only (renderer + container geometry +
boot default-view + the default-view INV-1 fixtures + Specs/ADR/README/showcase for
the default view).

DEFERRED (DO NOT TOUCH — flag as follow-up phase "g2-app UI 27px density rework"):
- All overlay panels: combat-tracker, inventory, spellbook, action-options, slot/
  target/template pickers, conc-modal, reaction-prompt, character-sheet, quick-action
  menu, wizard. Their renderers, geometry, and fixtures stay on the old grid.
- A real map-mode gesture toggle. For the default view, simply DO NOT mount/paint the
  raster map as the base. A full map mode (gesture-opened) is future work — note it in
  Specs §7.4 + ADR-0001 as deferred, do not build it.
- The GEST-01/ADR-0012 gesture-vocab sweep (`long=` → `qa=`) is Phase 20; the new R1
  hint row uses the over-scroll/double-tap vocabulary already agreed in the mockup
  (`R1: ^v scorri  tap ping  oo menu`) WITHOUT introducing `qa=` tokens that would
  collide with the Phase 20 sweep.
</objective>
