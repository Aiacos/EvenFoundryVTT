<objective>
Fix the LAST broken link of the live map chain. v0.1.10's `renderer.extract.pixels()` with no argument reads the WebGL MAIN framebuffer, which (with `preserveDrawingBuffer:false`) is only valid during the render pass. On an idle Foundry canvas the read returns ALL-ZERO frames — live-verified 2026-06-10: frames streamed at ~1Hz with maxG=0 while the player was idle, content appeared only during active canvas re-render.

Replace the capture with a deterministic RENDER-TO-TEXTURE viewport capture: create a viewport-sized PIXI `RenderTexture`, render the stage into it (the stage carries Foundry's pan/zoom transform, so the result IS the player viewport with fog-of-war), then extract from the texture. Keep the v0.1.10 no-arg read as a documented fallback for unit-test fixtures and exotic hosts where `PIXI.RenderTexture` is unavailable.

Purpose: glasses show a live, continuously-refreshed map regardless of canvas idle state — closes the final gap in the map-frame pipeline.
Output: updated `canvas-extractor.ts` (RT primary path + fallback), extended `canvas-extractor.test.ts` (CE-VP series), patch changeset.
</objective>
