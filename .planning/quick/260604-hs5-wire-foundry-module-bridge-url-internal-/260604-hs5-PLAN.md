<objective>
Link the Foundry module's outbound `/internal/delta` auth to the bridge deployment by adding two DM-editable world settings (`bridgeUrl`, `bridgeInternalSecret`) and making `getBridgeUrl()` / `getInternalSecret()` prefer them, falling back to the existing per-pair bearer-registry values.

Purpose: A real Forge-hosted Foundry world currently generates a RANDOM per-pair `internalSecret` that can never match the bridge's single static `process.env.EVF_INTERNAL_SECRET`, so every push gets 401 and real pairing is impossible. The DM must be able to enter the bridge URL + the bridge's actual `EVF_INTERNAL_SECRET` so pushes authenticate.

Output: Two visible world settings, settings-preferred resolution with backward-compatible bearer fallback, i18n catalog entries, unit tests, and a patch changeset.
</objective>
