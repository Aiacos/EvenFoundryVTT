Build a dev-only **Debug Console** for EvenFoundryVTT: a fast, gated observability +
command system spanning the full chain
`G2 glasses ← g2-app (phone WebView) ←WSS→ bridge ←socketlib→ Foundry`.

The developer (and Claude, via HTTP/WS) can:
- **Observe**: read a redacted bridge state snapshot + stream live inbound/outbound/tool/display events.
- **Inject**: push any envelope type to one or all sessions (bridge→client direction only).

- **Gesture**: simulate R1 gestures (`tap/double-tap/scroll-up/scroll-down/long-press`).
- **Display-mirror**: see what the glasses *would* render — g2-app's page rebuild / mount / destroy
  ops mirrored back into the debug feed (the high-value feed once physical glasses arrive).

Purpose: drive every function and watch the whole chain before/while the physical glasses arrive.

<security_model>
## CRITICAL — Debug endpoints mutate real state. Treat as a privileged dev backdoor.

These endpoints can **inject envelopes**, **dispatch real tools (Foundry writes)**, and
**simulate gestures**. They MUST be impossible to reach in production by default.

**Three-layer gate (all required):**

1. **Existence gate — `isDebugEnabled()`**: routes are NOT registered unless
   `process.env.EVF_DEBUG === 'true'`. When disabled, every path returns Fastify's
   default 404 (route literally absent). Production-safe default: even if `EVF_DEBUG`
   leaks `'true'`, layer 2 still blocks.
   - Rule: `isDebugEnabled()` = `process.env.EVF_DEBUG === 'true'`. In `NODE_ENV==='production'`
     it ADDITIONALLY requires `EVF_DEBUG_ALLOW_PROD === 'true'` (explicit double opt-in).
     Default production = OFF regardless of `EVF_DEBUG`.

2. **Auth gate — `EVF_INTERNAL_SECRET`**: every debug route (and the WS stream + dashboard)
   reuses the existing `secretsEqual` timing-safe check from `internal-delta.ts`. Missing/empty/wrong
   secret → 401. No bearer-token surface added; no new auth model.

3. **Redaction gate — T-02-01**: tokens NEVER appear in full in any snapshot, event, summary,
   or log. Only a `tokenHint` = first 6-8 chars + `…`. The ring buffer redacts on `push`.
   Pino redact list already covers logs; debug bus must redact independently (it bypasses pino).

**ADR-0011 compliance**: `/debug/dispatch-tool` routes through the SAME injected `dispatchToolFn`
(→ foundry-module via socketlib). The bridge NEVER calls `activity.use` directly. `/debug/inject`
and `/debug/simulate-gesture` only call `deltaEmitter.emitDelta` (bridge→client direction) — they
do NOT trigger client-side writes. No new socketlib handler is added (CI Gate 8 stays 17).

**Additive-only invariant**: when debug is disabled, the production message loop, delta-emitter,
and LayerManager behave EXACTLY as before. All taps are behind optional hooks/flags that are unset
in production — zero overhead (PerfProbe model).
</security_model>

## Debug API (registered only when `isDebugEnabled()`; all require `EVF_INTERNAL_SECRET`)

| Method | Path | Auth | Body | Response |
|--------|------|------|------|----------|
| GET | `/debug/console` (alias `/debug`) | secret | — | `200 text/html` dashboard (404 when disabled) |
| GET | `/debug/state` | secret | — | `200 { sessions: [{sessionId, tokenHint, locale, caps, lastSeq, createdAt}], replayBuffer:{size,lastSeq}, deltaEmitter:{currentSeq,connectionCount}, tokenCache:{size}, caches:{spell:bool, entity:bool}, metrics:{...summary} }` |
| GET | `/debug/events` | secret | query: `tail`(N), `type`, `direction`, `session` | `200 DebugEvent[]` (newest-last, capped by tail) |
| WS | `/debug/stream` | secret (query `?secret=` or header) | — | live `DebugEvent` JSON frames; unsubscribe on close |
| POST | `/debug/inject` | secret | `DebugInjectBody { type, payload, targetSessionId? }` | `200 { injected:true, seq, targetCount }` |
| POST | `/debug/dispatch-tool` | secret | `DebugDispatchBody { sessionId, toolId, idempotencyKey?, args }` | `200 { result: ToolInvokeResult, durationMs }` |
