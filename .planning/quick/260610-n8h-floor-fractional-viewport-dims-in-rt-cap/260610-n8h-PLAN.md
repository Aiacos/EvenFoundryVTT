<objective>
Fix the RT-path "pixel buffer length mismatch" skip that fires on every frame when
renderer.screen has fractional dimensions (live evidence: Forge client 2026-06-10,
screen = {width:2348.25, height:824.25} at devicePixelRatio 1.3333).

Root cause: vw/vh are derived as raw floats → RT.create receives fractional dims →
PIXI floors to integer internally → extract returns floor(vw)×floor(vh)×4 bytes,
but the byte-length guard computes expected = vw×vh×4 (fractional) → k ≈ 0.9998
(non-integer) → EVERY frame is skipped.

Fix: apply Math.floor at the derivation site so vw/vh are integers from the start.
RT.create, the byte-length guard, and the srcWidth/srcHeight returned by
acquireSourceBytes all see the same integer value → k = 1.000 exact → frames emit.

Purpose: Unblock the RT capture path on real Forge deployments with fractional DPR.
Output: 2-line production fix + 1 new test (CE-VP-8) + changeset.
</objective>
