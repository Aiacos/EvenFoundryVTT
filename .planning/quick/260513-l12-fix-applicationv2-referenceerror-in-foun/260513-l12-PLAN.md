<objective>
Fix the Foundry v13+ runtime `ReferenceError: ApplicationV2 is not defined` thrown
when `packages/foundry-module/dist/module.js` evaluates the `PairModal` class
declaration at module-load time. The class extends the bare global `ApplicationV2`,
which no longer exists in Foundry v13+ — the class lives at
`foundry.applications.api.ApplicationV2`. The crash interrupts the entire `init`
hook chain, so `game.settings.register('evenfoundryvtt', 'bearerRegistry', ...)`
never runs (manual console invocation succeeds, confirming the API is live).

Three coupled changes, all atomic per INV-4 (no half-updated runtime/type/test
states):

1. **Source fix**: `PairModal.ts` adds a local destructure of `ApplicationV2`
   from the namespace path so `extends ApplicationV2` (line 190) and
   `...ApplicationV2.defaultOptions` (line 216) bind to the real v13 class without
   any further code rewrite.
2. **Type-declaration fix**: replace the false-global `declare class ApplicationV2`
   in `src/types/foundry-globals.d.ts` (~lines 86–120) with a `declare namespace
   foundry.applications.api { class ApplicationV2 {...} }` block that matches the
   actual v13 runtime. Remove the unused `declare class Application` (zero dead
   code, INV-4) — confirmed unused in `src/` (only test files reference an
   `ApplicationStub` local class, never the ambient `Application` type).
3. **Test stub fix**: replace `vi.stubGlobal('ApplicationV2', ApplicationV2Stub)`
   in 4 test files with `vi.stubGlobal('foundry', { applications: { api: {
   ApplicationV2: ApplicationV2Stub } } })`. The existing `ApplicationV2Stub`
   class bodies are preserved unchanged — only the binding mechanism changes.

Purpose: restore Foundry v13/v14 boot for `evenfoundryvtt@0.1.2`-installed worlds.
Manual smoke is documented in SUMMARY.md as next-step (user must reload module in
Foundry); automated coverage is the existing vitest suite passing with the new
namespace-stub pattern.

Output: source + types + tests committed in one commit per INV-4 atomic-update
discipline; `pnpm typecheck`, `pnpm test`, `pnpm build`, `pnpm lint:ci` all green.
