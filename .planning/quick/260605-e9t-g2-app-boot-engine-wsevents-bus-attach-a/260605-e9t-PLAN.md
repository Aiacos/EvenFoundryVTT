<objective>

The HUD therefore boots showing placeholders (PF …, CA —); only a LATER live delta renders correctly.

Fix (last-value replay): refactor `createWsEventBus` so it attaches ONE persistent `message` listener at bus creation, caches the last payload per channel (`lastByChannel: Map<string, unknown>`, keyed by envelope `type`), and makes `subscribe()` immediately replay the cached last value before registering for future envelopes. Create the bus right after WS open and BEFORE `performCapabilityHandshake` so its listener is live when the on-connect delta arrives. The Phase 10 D-Area1 `seqTracker.observe()` + `perfProbe.mark()` hot-path behavior is preserved exactly.

Purpose: HUD renders real character data on first connect — closes the WS-DATA-path timing gap left open after d0v/dog/qm0.
Output: Refactored `createWsEventBus` (now exported + persistent + replay) + early bus creation in `_bootEngineCore` + a dedicated TDD test file. g2-app only.
