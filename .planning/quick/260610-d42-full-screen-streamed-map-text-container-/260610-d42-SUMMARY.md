### Gap 1 — bridge not injected into CanvasStatusHudLayer in production (CLOSED)

**Root cause:** `boot-engine-core.ts` step 10 constructed `CanvasStatusHudLayer` without passing `bridge`:

```ts
// BEFORE (broken)
const canvasStatusHud = new CanvasStatusHudLayer({ wsEvents: wsEventBus });
// AFTER (fixed)
const canvasStatusHud = new CanvasStatusHudLayer({ wsEvents: wsEventBus, bridge });
```

Because `CanvasStatusHudLayerOpts.bridge` is optional, the missing argument caused `this._bridge === undefined` at runtime. The guard `if (this._bridge !== undefined)` in `_onDelta` silently skipped every `bridge.textContainerUpgrade` call — meaning the native `hud-status` text container (id=5) was never updated in production. Tests passed because they inject `bridge` explicitly into `CanvasStatusHudLayer` opts; the integration boot tests never asserted the bridge was forwarded.

**Fix:** Single-character change in `boot-engine-core.ts` line 823. `bridge` is already in scope (assigned from `bridgeFactory()` at step 2, used for `mapBase`, `idleInfill`, `statusHud`, `hudDeltaDriver`, `layerManager`, etc.).

**Regression test added:** `CR-01d` in `boot-engine-glyph-fallback-mount.test.ts` — boots in canvas mode (default), fires a valid `character.delta` WS envelope through the mock socket, awaits microtasks for the fire-and-forget bridge call, asserts `bridge.textContainerUpgrade` call count increased. This is the minimal integration-level proof that the bridge reaches `CanvasStatusHudLayer._onDelta`.

**Files modified:**
