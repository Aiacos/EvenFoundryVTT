<objective>
Encode the PROVEN manual EvenHub-simulator dev/test recipe (live-verified across the 260605-* session: see STATE rows e9t/flv) into a single repeatable harness so testing the g2-app HUD is effortless: `pnpm sim start`.

Purpose: The HUD render loop currently requires a 6-step manual dance (bridge no-auth, then seed roster + character snapshots, then vite no-auth, then xvfb+GTK EvenHub simulator, then screenshot/console verify, then fuser teardown). It works, but only the author knows the exact env vars and footguns. Formalize it.

Output:
- `scripts/sim.sh` — start/stop/seed/shot subcommands, OS/DISPLAY detection, idempotent start, friendly stdout.
- `scripts/sim-fixtures/` — committed, schema-valid JSON: `roster.json` + one `character.delta` per PC.

- Small edits: root `package.json` `"sim"` alias + README pointer + `.gitignore` runtime dir.

No changes to bridge / g2-app / shared-protocol source. Additive operational tooling only.

Known footguns this harness MUST encode (verified-correct):
(a) the GTK env block is mandatory in headless mode or the simulator crashes with glycin-loaders;
(b) teardown uses `fuser -k PORT/tcp`, NOT `pkill -f vite`/`pkill -f evenhub` (those self-match the script's own command line);
(c) the bridge cache is IN-MEMORY, so every (re)start must re-seed;
(d) the `?actor=` query must be appended to the simulator's target URL, not added afterward.
