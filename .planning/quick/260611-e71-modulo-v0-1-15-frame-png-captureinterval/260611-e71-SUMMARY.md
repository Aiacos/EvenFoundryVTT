## What Was Built

### shared-protocol (FRAME-PNG-01)
- `FramePngSchema`: `{sceneId, width (20-576), height (20-288), pngB64, ts}` — full-screen layout B bounds
- `FRAME_PNG_TYPE = 'frame_png' as const` — wire type discriminator
- `type FramePng` — TypeScript type alias
- `FramePixelsSchema` retained — back-compat export unchanged
- 9 accept/reject test cases in `frame-png.test.ts`

### foundry-module v0.1.15 (FRAME-PNG-02)
- **PNG encode pipeline** in `canvas-extractor.ts`:
  1. Fit-downscale + letterbox (unchanged)
  2. Optional contrast normalization (unchanged)
  3. Rec.601 luma: `luma = round(0.299*R + 0.587*G + 0.114*B)` → `Uint8Array`
  4. Build R=G=B=luma RGBA for UPNG
  5. `UPNG.encode([rgbaLuma.buffer], w, h, 0, undefined, true)` — verified recipe
  6. Base64 via `Buffer.from().toString('base64')` (Node) / `btoa+chunk` (browser)
- **FNV-1a 32-bit hash** over luma array — skip identical frames (no POST)
- **Leading+trailing throttle** (THROTTLE_MS=200ms) replaces trailing-only debounce
- **Live captureIntervalMs gating** via TICK_MS=100ms setInterval — DM changes cadence live
- **`captureIntervalMs` world setting** (Number, scope:world, config:true, restricted:true, default:250, range:100-5000, step:50) + `getCaptureIntervalMs()` live getter
- **upng-js bundled** in tsup `noExternal` — codec available in Foundry data folder
- Version bumped to 0.1.15 in package.json + module.json
