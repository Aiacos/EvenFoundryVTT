**How app.json whitelist is handled without committing private origins:**
The committed `app.json` keeps safe placeholder whitelist entries (`https://bridge.REPLACE-WITH-YOUR-ORIGIN.example` and `https://plugin-host.REPLACE-WITH-YOUR-ORIGIN.example`). Each self-hoster runs `node deploy/sync-app-whitelist.mjs` (reads `EVF_BRIDGE_HOST` + `EVF_PLUGIN_HOST` from `deploy/.env`, writes real HTTPS origins into the whitelist, exits 1 if values are missing or still `*.example.com`), then `corepack pnpm --filter @evf/g2-app pack:ehpk`. The resulting `.ehpk` contains their private origins and is never committed. `git diff packages/g2-app/app.json` is empty at task end.
